# App Store 제출 체크리스트 & 설문 정답지

업로드된 빌드(이미 완료)에 이어, App Store Connect 웹에서 채워야 할 항목과 **그대로 선택하면 되는 정답**을 정리했습니다. 이 앱은 데이터를 전혀 수집하지 않으므로 대부분 "없음"입니다.

## 0) 먼저: 빌드 처리 확인
App Store Connect → 앱 → **TestFlight**(또는 버전의 "빌드" 섹션)에서 업로드한 빌드가 "처리 중 → 사용 가능"이 될 때까지 기다립니다(보통 5분~1시간).

## 1) 스크린샷  ✅ 자동 생성됨
- 파일: `store/screenshots/01-welcome.png`, `store/screenshots/02-tabs-korean.png`
- 크기: **2560 × 1600 px** (App Store 허용 macOS 사이즈). 최소 1장, 최대 10장.
- 위치: 버전 페이지 → "App 미리보기 및 스크린샷" → macOS에 드래그.
- 더 만들려면: `./store/make-screenshots.sh` 다시 실행.

## 2) 앱 개인정보(App Privacy)  → "데이터 미수집"
경로: 앱 → **앱 개인정보** → 시작.
- "이 앱에서 데이터를 수집합니까?" → **아니요 (Data Not Collected)** 선택.
- 끝. (네트워크·분석·추적 전혀 없음.)
- **개인정보 처리방침 URL(필수)**: `store/privacy-policy.html`을 호스팅한 주소 (아래 4번).

## 3) 연령 등급(Age Rating)  → 전부 "없음" → 4+
경로: 앱 → (버전 아님) 앱 정보/연령 등급 설문.
- 모든 콘텐츠 항목(폭력, 성적 콘텐츠, 욕설, 약물, 도박, 공포 등) → **없음/해당 없음**.
- 무제한 웹 접근: **아니요**. 도박/콘테스트: **아니요**.
- 결과: **4+ (전 연령)**.

## 4) URL 두 개 (필수: 지원 URL, 개인정보 URL)
둘 다 `https://`여야 합니다. 가장 쉬운 무료 호스팅 = GitHub Pages:
1. 이 저장소를 GitHub에 push (public).
2. GitHub → 저장소 → **Settings → Pages** → Source: `main` 브랜치, 폴더 `/ (root)` 또는 `/docs`.
3. `store/privacy-policy.html`을 저장소 루트(또는 docs)로 복사해두면 →
   `https://<사용자명>.github.io/<저장소명>/privacy-policy.html` 로 열림.
- **지원 URL**: 저장소 README 주소(`https://github.com/<사용자명>/<저장소명>`)로 충분.
- **개인정보 처리방침 URL**: 위 Pages 주소.

## 5) 가격 및 사용 범위
- 가격: **무료(Free)** / 모든 국가 또는 원하는 국가 선택.

## 6) 그 외 버전 정보
- 설명/키워드/프로모션/부제/이름: `store/listing-ko.md`, `store/listing-en.md`에서 복붙.
- 카테고리: 기본 **생산성**.
- 저작권: `© 2026 Min-Gul Kim`.
- **수출 규정(암호화)**: 프로젝트에 `ITSAppUsesNonExemptEncryption = NO`를 넣어둬서 보통 질문이 안 뜹니다. 뜨면 "암호화 사용 안 함/면제" → **아니요**.
- **로그인 필요?** 아니요 → 심사용 데모 계정 불필요(앱에 로그인 없음). "로그인 필요" 체크 해제.
- **콘텐츠 권한**: 제3자 콘텐츠 없음.

## 7) 제출
버전 페이지에서:
1. 처리된 **빌드 선택**
2. 스크린샷·설명·URL·연령등급·가격 확인
3. **심사를 위해 제출(Add for Review → Submit)**
4. 심사: 보통 1~3일. 리젝되면 사유(이름 충돌, URL 누락 등)만 고치면 됩니다.

## 자주 막히는 포인트
- **이름 충돌**: "Notepad"/"메모장" 단독은 이미 있을 수 있음 → 고유 이름 필요(listing 파일의 결합형 참고).
- **개인정보 URL 누락** → 4번 호스팅 필수.
- **스크린샷 사이즈** → 반드시 허용 사이즈(여기 2560×1600은 OK).
